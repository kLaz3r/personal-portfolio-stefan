## Project Overview

Modern personal portfolio website built with Next.js 16.2.1, TypeScript, and Tailwind CSS 4.2.2, featuring animated UI, project showcase, and automated photo gallery management.

## Development Commands

```bash
# Install dependencies
npm install

# Start dev server with Turbopack (uses port 3000)
npm run dev

# Build production bundle
npm run build

# Start production server
npm start

# Run linting
npm run lint
```

## Project Structure

- `src/app/` - Next.js App Router structure (pages at page.tsx, global CSS at globals.css)
- `src/components/` - Reusable React components (all have relative imports to `../data` and `../contexts`)
- `src/data/` - Static data files (projects.ts and automatically-generated photos.ts)
- `src/contexts/` - React context providers (ThemeContext for dark/light mode)
- `public/` - Static assets including images, SVGs, and mockup screenshots
- `public/images/` - Source images for the photo gallery (for Python automation)

## Architecture Details

### Next.js Configuration

- Uses App Router with TypeScript strict mode
- Image optimization disabled via `next.config.ts` (image optimization redirects disabled)
- Custom allowed dev origin: `192.168.3.3` configured for local network testing
- Path aliases: `@/*` maps to `./src/*` (can use `@/components/ComponentName`)

### Styling System

- Tailwind CSS 4.2.2 with PostCSS handling compilation
- Global styles in `src/app/globals.css` with CSS variables for fonts and halftone patterns
- Montserrat font from Google Fonts loaded in layout.tsx
- Native CSS transitions preferred over library animations for performance

### Data Management

**Projects** - Manually maintained in `src/data/projects.ts` with structure:
```typescript
title: string
description: string
imageUrl: string (must be in /ss/ directory)
tags: string[]
liveUrl: string? 
repoUrl: string?
```

**Photos** - Auto-generated by Python script (see Automation section)

### State & Theme Management

- ThemeContext handles dark/light mode with localStorage persistence
- Initial theme determined by: saved preference → system preference → defaults to light
- Theme class applied to `<html>` element for Tailwind dark mode
- Theme toggle available via Navbar component

### Components Structure

- **Layout Components**: Footer, Navbar, ScrollProgressBar, RootLayout
- **Project Showcase**: ProjectCard, ImageCarousel (together in carousel on homepage)
- **Photo Gallery**: NextJsImageLightbox (wrapper for lightbox) 
- **Icons**: Custom SVG icons in ApertureIcon, CodeIcon, QuestionMarkIcon, SwatchIcon
- **Background**: HalftoneBg generates complex SVG patterns (do not modify lightly)

## Automation: Photo Gallery

Photo gallery data is auto-generated via Python script:

```bash
# Install dependencies first
pip install Pillow

# Run script from project root
python generate-image-array.py
```

**What it does**: Scans `public/images/` → calculates aspect ratios → generates `src/data/photos.ts`

**When to run**: After adding/removing images in `public/images/` or when photos appear incorrectly sized

## SEO & Performance

- Extensive SEO metadata configured in RootLayout (Open Graph, Twitter cards, Schema.org structured data)
- Analytics: Umami script configured at `umami.stefann.duckdns.org`
- Sitemap and robots.txt pre-configured
- Header scroll progress indicator for UX feedback

## Dependencies to Know

- **motion** - Primary animation library (do not add framer-motion, use motion instead)
- **react-photo-album** + **yet-another-react-lightbox** - Photo gallery masonry layout and lightbox
- **react-icons** - Icon library (currently using various icons across components)

## Code Conventions

- All React components follow FC pattern with explicit return types
- Component props use TypeScript interfaces
- Relative imports preferred for data files (`../data/projects` not `@/data/projects`)
- Props follow the type alias (`type Props = { ... }`) pattern
- Components use arrow functions (e.g., `export const ComponentName = ({ prop }): JSX.Element => {`)
- Image imports use Next.js built-in optimization (same domain only)
